What is the Internet?

  • Based upon a networking infrastructure that allows hardware devices to connect to eachother
    • Servers, Desktop PCs, Mobile devices etc.
    • Based on common standards - notably the TCP/IP protocol suite.

  • Protocols specify how the transactions are carried out
    • They say how communication is initiated and maintained.
    • Type, size and structure of the data to be sent and recieved.
    • Enable the services running over the internet.

  • Software clients (and servers) allow users and devices to access and use internet services
    • Using centralised servers or P2P (Peer-2-Peer) systems.

Web Clients

Web Clients

  • Web Clients are pieces of software which are used to create and send HTTP requests, and handle the responses
  • Mosaic kick-started mainstream interest - then came Internet Explorer and Netscape
  • Many of the common browsers share common layout engines e.g. Trident, Gecko, KHTMl/Webkit etc.

Simple DNS Request

The diagram shows how a simple DNS (Domain Name System) request works, between a client and a remote server.

  1. Client Request
  2. DNS Query to a DNS server, this is where your browser asks a remote server called a DNS Server, which then responds with the IP address where the target server is located.
  3. The DNS server responds with the target IP
  4. The client then opens a 2-way TCP connection with the target web server
  5. A protocol request is sent
  6. A protocol reply is recieved

Web Servers

Web Servers

  • "Web Server" refers to software not a physical computer, one single computer can run multiple servers
  • Server listens on an open network port for incoming HTTP requests and responds accordingly
  • Original specs from CERN released in 1991
    • Until 1995 dominant server was the "NCSA server"
    • NCSA spawned the Apache project (#1 by the end of 1996)
    • Apache still dominates the server market

Image from Netcraft

Typical Web server features

Path translation - Locate resource specified

Virtual hosting - Run multiple websites from one server

Access restriction - Control who can access the resources>

Log access & errors - Record what's going on (and what went wrong!)

Encrypted operation - Protect sensitive data in transit

Load balancing - Manage traffic and processing resources

Server scripting - Run external programs and applications

Web architechture

HTTP, URIs and URLs

Key web architechture components

  • HTTP/1.1 HyperText Transfer Protocol
    • Format and semantics of request/response messages
  • URI: Uniform Resource Identifier
    • Formatted string that identifies a resource
  • URL: Uniform Resource Locator
    • Subset of URI's that, in addition to locating a resource, provides a means of locating the resource by describing its network location
  • HTML/XHTML: HyperText Markup Language and Extensible HTML
  • DNS: Domain Name System
  • TCP/IP: Internet Protocol Suite

History

1991

CERN introduces the World Wide Web to the public

1990

Tim Berners-Lee, at CERN develops HTML.

1989

World.std.com becomes first provider for dial-up access for the internet

1987

Number of hosts on the internet exceeds 20,000. Cisco ships its first router

1985

Symbolics.com, the site for Symbolics Computer Corp., becomes the first registered domain.

1983

Domain Name Service (DNS) creates .edu, .gov, .com, .mil, .org, .net and .int system for websites

1979

USENET is created to host news and discussion groups

1972

TCP and IP emerge as the protocol for ARPANET. This becomes the standard definition for the internet

1976

Queen Elizabeth II sends her first email

1974

Vinton Cerf and Bob Kahn publish a paper detailing the design of TCP

1973

Global networking becomes a reality as the University College of London and the Royal Radar Establishment (Norwary) connect to ARPANET, term Internet is born.

1969

Stanford Research Institude (SRI), sends first message "LO", was an attempt to send "LOGIN" from the University of Utha. The system crashed after "LO"

1965

Two computers at MIT Lincoln have communicated with each other using packet-switching technology
